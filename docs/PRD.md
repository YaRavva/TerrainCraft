Техническое задание на разработку веб-приложения для генерации 3D-объектов
1. Общая информация

Название проекта: 3D Tile Generator (или аналогичное, например, "LandscapeDreamer" по аналогии с CityDreamer).
Цель проекта: Разработать веб-приложение, позволяющее пользователям генерировать 3D-объекты в форме квадратных или прямоугольных тайлов (плиток), обеспечивая их бесшовное соединение в единый coherent ландшафт. Генерация осуществляется на основе текстового описания (промпта) или входного изображения. Приложение должно вдохновляться проектами вроде CityDreamer (2024), но без обязательного использования алгоритма Wave Function Collapse (WFC), с акцентом на procedural generation для создания реалистичных или слегка мультяшных сцен, включая элементы такие как дома, парки, реки, озера, дороги и деревья.
Целевая аудитория: Разработчики игр, дизайнеры ландшафтов, художники, студенты (например, для моделирования университетских кампусов), энтузиасты 3D-моделирования.
Тип приложения: Веб-приложение (frontend + backend), доступное через браузер, с возможностью экспорта сгенерированных моделей.
Бюджет: бесплатно

2. Функциональные требования
2.1. Входные данные

Поддержка двух режимов генерации:
По текстовому описанию: Пользователь вводит промпт (например, "университетский кампус в осеннем сезоне с домами, парками и рекой").
По изображению: Пользователь загружает изображение (layout или референс, например, bird's-eye view), которое анализируется для генерации 3D-модели.

Дополнительные параметры:
Выбор стиля: Реалистичный, слегка мультяшный (cartoonish), hand-drawn.
Вариации: Поддержка генерации в разных "сезонах" или стилях (весна, лето, осень, зима, или аналогично четырем вариантам как в описанных проектах).
Размер тайла: Квадратные или прямоугольные (настраиваемые размеры, например, 256x256 единиц).
Количество тайлов: Автоматическое или ручное указание для соединения в больший ландшафт (например, 4x4 сетка).


2.2. Процесс генерации

Генерация тайлов: Создание индивидуальных 3D-объектов (тайлов) с элементами ландшафта: дома (buildings), парки (greenlands), реки/вода (water bodies), дороги, деревья.
Бесшовное соединение: Механизм для seamless интеграции нескольких тайлов в единый ландшафт без видимых швов. Использовать compositional generative models (как в CityDreamer), с правилами соседства (connectors, neighbor rules) для coherent структуры. Обеспечить unbounded (бесконечную) генерацию для больших сцен.
Алгоритмы: Избегать WFC; использовать GAN-like модели, diffusion models или procedural rules (например, на базе CGA из CityEngine). Интеграция с открытыми моделями вроде Stable Diffusion для 3D или аналогичными (TripoSR, InstantMesh для текст-to-3D).
Примеры сценариев: Генерация университетского кампуса в четырех сезонах (весна с цветами, лето с зеленью, осень с листьями, зима со снегом), с бесшовным ландшафтом из тайлов.

2.3. Выходные данные

Визуализация: Интерактивный 3D-viewer в браузере (на базе Three.js или Babylon.js) для просмотра сгенерированного ландшафта.
Экспорт: Форматы OBJ, GLTF, STL для скачивания. Опционально: Экспорт в Unity или другие движки.
Галерея: Хранение и просмотр предыдущих генераций (сессионное или с аккаунтом).

2.4. Пользовательский интерфейс (UI/UX)

Frontend:
Главная страница: Форма для ввода промпта/загрузки изображения, выбор параметров (стиль, сезон, размер).
Просмотр: 3D-рендер с зумом, ротацией, панорамой.
Галерея: Список сгенерированных моделей с превью.

Аутентификация: Опциональная регистрация для сохранения проектов.
Responsive дизайн: Адаптация под мобильные устройства.

2.5. Backend

Обработка запросов: API для генерации (REST или GraphQL).
Интеграция моделей: Использование предобученных моделей (например, из Hugging Face) для text-to-3D или image-to-3D.
Хранение: База данных (PostgreSQL) для пользовательских данных; облачное хранилище (S3) для моделей.

3. Нефункциональные требования

Производительность: Время генерации одного тайла — до 30 секунд; полного ландшафта — до 2 минут (использовать GPU-ускорение, если возможно).
Масштабируемость: Поддержка множественных пользователей; облачный деплой (Vercel).
Безопасность: Защита от вредоносных промптов; валидация загружаемых изображений.
Доступность: Открытый исходный код (MIT-лицензия), репозиторий на GitHub (вдохновлено CityDreamer 2024).
Технологии:
Frontend: React.js, Three.js.
Backend: Node.js
База данных: Supabase
Деплой: Vercel

Тестирование: Unit-тесты для генерации; UI-тесты; проверка бесшовности на примерах (кампус в сезонах).
Документация: README с инструкциями; API-доки; примеры использования.

4. Риски и зависимости

Зависимости от внешних моделей: Если модели (как CityDreamer) недоступны, использовать альтернативы (OpenAI Shap-E, Google DreamFusion).
Риски: Высокие вычислительные требования; юридические аспекты (авторские права на генерируемый контент).
Дополнительно: Интеграция с датасетами вроде OSM/Google Earth для реализма, как в CityDreamer.
